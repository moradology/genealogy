### 0.2 House rules (binding, from the owner)

1. **Evidence honesty.** Conflicts are documented, never hidden. Conjecture is always
   visually distinct from documented fact. Negative results (searched-and-not-found)
   are recorded, not discarded.
2. **No living people on the page. Ever.** Watch for this in census finds (e.g. a
   1937-born grandchild in a 1950 census is very plausibly alive).
3. **Self-contained.** index.html makes zero external requests. Fonts, images, map
   data all inline. The acceptance spec enforces this by blocking all routes.
4. **Both themes.** Light and dark must both be designed, and the user's toggle
   (`data-theme` on `<html>`) must beat the OS preference in both directions.
5. **Code style:** vanilla JS functions, no frameworks, no try/catch. Python via
   `uv run` only (never bare `python`/`pip`), stdlib-preferred, no try/except.
6. **Sensitive material with dignity.** Example: Medical Lake WA = Eastern State
   Hospital (psychiatric). State documented fact plainly; no speculation, no drama.
7. **Byte ceiling (the "treaty"):** index.html never exceeds 320KB (327,680 B).
   Current: ~228.7KB. Spec budgets (S9 total <251,546; S10 fonts <45,153; S11 map
   paths <21,739) are measured+10%; raise them only deliberately, in the same commit,
   with the byte cost stated in the commit message.
8. **Append-only structures:** Docket case numbers (case.01‚Ä¶), source display codes
   (s1‚Ä¶s126‚Ä¶), corrigenda list. Never renumber, never delete; supersede.

### 0.3 Id grammar (the "treaty" naming rules)

- `person.<slug>` ‚Äî lowercase `[a-z0-9_]+` only. Dots in ids mean: address them with
  `getElementById(...)` / `href="#..."`, never bare `querySelector('#person.x')`.
- `case.NN` ‚Äî two digits, the Docket.
- `event.<family>.<description>.<date>` ‚Äî geojson feature ids double as event ids.
- `src.<title-slug>.<sha1-of-url-8hex>` ‚Äî ledger ids.
- `place.<slug>` ‚Äî defined ONLY in the geojson top-level `place_registry`.
- Ahnentafel numbers per anchor chart: prefixes Z- (Doyle), M- (Evelyn), D- (Bill),
  C- (Donna); person 1 = the anchor, father of n = 2n, mother of n = 2n+1.
- `tools/check_refs.py` enforces closure: every reference must resolve to a defined
  id. Run the gate after touching any id.

### 0.4 The validation protocol (non-negotiable for research claims)

Nothing from the research sweep (or any external claim) lands in `index.html`, the
geojson, or a reasoning trace until an agent has **independently re-fetched the cited
source and matched the quote verbatim**. Reporting agents have mislabeled sources
before (a grave-marker photo was once reported as a portrait). Confidence tiers in
the intake files are the *reporting agent's* grade of its own source access:

- `verified-primary` ‚Äî agent fetched a primary/near-primary record and quotes it
- `verified-index` ‚Äî agent fetched an index or derivative
- `compiled` ‚Äî trees/secondary compilations; treat as hearsay until proven
- `speculative` ‚Äî hypothesis, clearly flagged

Landed facts go to: person rows + Docket cases (+ geojson + ledger as needed).
Searched-and-not-found goes to the negative-evidence register (Slate 1 builds it).
Corrections to already-published content go to the dated corrigenda list.
